<State Vs Props>
- State는 변경가능한데 소유한 component에서 setState method를 통해 가능
- setState는 change value, notification, and rendering 함. 비동기 동작임.
- Props 는 부모에서 자식으로 전달되는데 자식에서 직접 변경불가 (객체를 전달하면 가능)
  부모 state를 변경하려면 부모에서 전달된 부모 callback 함수를 사용해야 함.
- State 보관은 부모 component나 주요 component에서 집중하는 것이 좋음.
- State 객체가 복잡하면 불변성 library를 사용하는 것이 좋음: immer
- React의 최대 장점은 fast rendering인데 불변성 관리 최적화를 꼭 해야 됨 (?)
- State는 관리가 필요한 중요한 데이타인데 중요함의 기준은  >>y> controlled data
   1) 여러 component에서 사용하는 data이거나,  >>y> sharing data
   2) 추적이 필요한 data  >>y> tracking data
  임.


<Unique Key generation>
1) UUID   : npm uuid
2) ShortID (8 byte): npm shortid
3) ObjectID (12 byte, used in MongoDB): npm install objectid
    12 = 4 (timestamp) + 3 (computer ip) + 2 (process id) + 3 (Incremental number)

<배포>

- source-map 은 배포하지 않도록 한다
- react 개발자 도구를 비활성화해야 한다: component tree 구조를 보지 못하도록 한다.
- npm rimraf는 파일을 remove하는 기능을 제공합니다
yarn add -D rimraf cpy-cil
그 후 package.json 에
  scripts: {
  "build:nomap": "react-scripts build && rimraf build/**/*.map && rimraf d:/www/* && cpy ** d:/www/ --cwd=build --parents",
}

build후 build 디렉토리 아래에서 map을 삭제하는것입니다.
